# Trap Case V2 Eval Plan

## Purpose

This fixture replaces the ambiguous triangle-wave trap with an executable quality oracle and a file large enough to trigger RepoLens' default `8000` token threshold.

## Fixture

```text
tests/fixtures/trap-case-v2/
  PROMPT.md
  package.json
  src/audio-engine-large.ts
```

`src/audio-engine-large.ts` is generated by:

```bash
npm run trap:v2:generate
```

The target file is intentionally larger than 8k estimated tokens.

## Quality Oracle

Run the gate against any completed eval workspace:

```bash
npm run trap:v2:gate -- <workspace>
```

Pass condition:

```text
Oscillator(1 Hz), triangle waveform, gain 1, sampleRate 4
must produce: 0, 1, 0, -1, 0
```

This avoids scoring by comments or by whether a specific formula was used.

## Recommended Matrix

```text
off x3
warn@8000 x3
strict@8000 x3
```

Optional lower-threshold comparison:

```text
strict@3000 x3
```

Optional adaptive comparison:

```text
REPOLENS_INCLUDE_ADAPTIVE8000=1
adaptive@8000 x3
```

For this single-full-read fixture, `adaptive@8000` is expected to behave closer to `warn@8000`: it warns first and blocks only a later repeated large full read.

## Standard Prompt

Use the contents of `tests/fixtures/trap-case-v2/PROMPT.md`.

## Runner

Preview the planned matrix without calling an external model:

```bash
npm run trap:v2:real-eval -- --dry-run
```

Run the matrix only after explicitly approving external model disclosure:

```bash
REPOLENS_ALLOW_EXTERNAL_MODEL=1 npm run trap:v2:real-eval
```

Optional environment variables:

```text
REPOLENS_EVAL_RUNS=2
REPOLENS_EVAL_MODEL=dee-seek/deepseek-v4-pro
REPOLENS_INCLUDE_STRICT3000=1
REPOLENS_INCLUDE_ADAPTIVE8000=1
REPOLENS_TRAP_ROOT=<tmp-root>/repolens-trap-v2-manual
```

## Safety Boundary

Run every model invocation in a fresh `<tmp-root>/repolens-trap-v2-<timestamp>/workspaces/<run-name>` copy of the fixture. Do not run the model inside the RepoLens repository.
